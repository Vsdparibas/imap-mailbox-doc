### new Imap(config)
Attendre la méthode run() avant quelconque action sur cet objet.


| Param | Type | Description |
| --- | --- | --- |
| config | <code>[ImapConfig](/docs/documentation/imap-config)</code> | Configuration de Imap |

<a name="Imap+run"></a>

### imap.run()
Lance le service IMAP
Tu ne dois faire aucune action sur l'objet Imap avant d'avoir effectué cette méthode.

**Type**: méthode d'instance de [<code>Imap</code>](#Imap)  

<a name="Imap+on"></a>

### imap.on(event, callback)
Utilise le callback quand un évènement est émis.

**Type**: méthode d'instance de [<code>Imap</code>](#Imap)  

| Param | Type | Description |
| --- | --- | --- |
| event | `'newMail'` / `'loadedMail'` / `'deletedMail'` | Évènement à écouter |
| callback | `(mail) => void` | Fonction callback à éxecuter quand l'évènement est émis |

<a name="Imap+deleteMails"></a>

### imap.deleteMails(mailboxPath, toDelete) ⇒ <code>boolean</code>
Supprimer des mails depuis leur uid ou l'objet [`Mail`](/docs/documentation/mail).

**Type**: méthode d'instance de [<code>Imap</code>](#Imap)  
**Retourne**: <code>boolean</code> - `true` si les mails ont été supprimés

| Param | Type | Description |
| --- | --- | --- |
| mailboxPath | <code>string</code> | Nom de la boîte mail où supprimer les mails |
| toDelete | <code>[UidsList](/docs/documentation/uids-list)</code> | Mails ou uids à supprimer |
